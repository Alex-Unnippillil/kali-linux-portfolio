Exit code 126 — Command invoked cannot execute

The shell located the command but could not run it. Most often this means the file is missing the executable bit or is built for another architecture.

Fix permissions

  1. Inspect the file with ls -l script.sh.
  2. If the execute bits (x) are missing, run chmod +x script.sh.
  3. Re-run the script: ./script.sh.

Other causes

  • Windows line endings. Convert with dos2unix script.sh.
  • Binary built for a different CPU. Download the version that matches your OS.
  • Missing interpreter line. Ensure scripts start with a shebang like #!/bin/bash.

Verify the fix

  • Use file script.sh to confirm the interpreter.
  • Run bash -n script.sh to check for syntax errors before execution.
  • Commit the corrected file so the executable bit is preserved in version control.
