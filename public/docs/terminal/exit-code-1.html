Exit code 1 — General error or missing resource

Exit status 1 is the shell's catch-all for commands that ended with a recoverable problem. Missing files, invalid arguments, or permission issues are the usual suspects.

Checklist

  • Double-check the spelling and capitalization of files, directories, and flags.
  • Run ls -lah to verify the target exists and you can access it.
  • Inspect recent commands with history to reuse a known-good invocation.
  • Use man <command> or --help to confirm the correct options.

Why this happens

Unlike specific error codes, exit 1 is generic. Tools return it when the error doesn't map to a dedicated status. Because of that, reviewing the text of the error message is crucial to narrow things down.

Next steps

  • Re-run the command with -v or --verbose if available.
  • Confirm permissions with stat <file> when working on shared directories.
  • Use strace or script to capture a failing run for later review.
