Exit code 130 — Interrupted by user

Pressing Ctrl+C sends SIGINT, ending the process with exit status 130.

When it's expected

  • Stopping a command that was taking too long.
  • Canceling a mis-typed command before it causes changes.
  • Exiting a foreground program so you can run something else.

Resume work safely

  • Re-run the command if you canceled it accidentally.
  • Use Ctrl+Z followed by bg to background long-running jobs instead of terminating them.
  • Wrap critical tasks in nohup, screen, or tmux so they survive interrupts.

Automate cleanup

Shell scripts can trap SIGINT to run cleanup logic: 

trap 'echo "Cleaning up..."; rm -f /tmp/app.lock' INT
./run-long-task.sh
